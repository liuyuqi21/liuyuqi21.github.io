并发扣款下的数据一致性

撰写于 2020-01-19 分类 问题 标签 分布式

场景

还款：

  1. 用户手动还款
  2. 系统自动还款
  3. 商户后台扣款

问题

出现多比代扣

原因

并发

解决

  1. 数据库使用 orderId 来标识订单的唯一性，支付状态为支付中则不允许支付
  2. java 中加锁：APP 与 商户后台部署在不同机器，进程内的锁无法解决问题
  3. 分布式锁

分布式锁

  1. redis/zk 实现悲观锁
  2. 乐观锁
  3. sql 语句实现
insert into XXX SELECT XXX FROM DUAL where not exists(select id from cl_borrow_repay where borrow_id = #{borrowId,jdbcType=BIGINT})
